What is Missingarr?

Missingarr is a lightweight, self-hosted tool that connects to your Sonarr (TV shows) and Radarr (movies) instances and automatically searches for missing content and quality upgrades at configurable intervals.

It runs as a single Docker container with no external database required. Configuration is persisted in a SQLite file in the /data volume.

Search Order Explained
  • Random — Items are shuffled before each run. Good for even distribution across your library.
  • Smart — 50% of slots go to items from the last 30 days (recent), 30% random, 20% oldest. Balances new releases with backlog.
  • Newest First — Recently added or released items are searched first.
  • Oldest First — Items waiting longest are searched first. Useful for clearing old backlogs.
Missing Mode (Sonarr only)
  • Episode — Searches for individual missing episodes. Most granular.
  • Season Packs — Searches for entire seasons as a pack. Useful if your indexers provide season packs.
  • Show Batch — Triggers a search for the entire series at once. Use with caution — can be slow.
  • Smart — Automatically chooses Season Pack if ≥50% of a season is missing, otherwise falls back to Episode search.
Rate Limiting

Rate limiting prevents your indexers from being overwhelmed with search requests.

  • Rate Window — The rolling time window in minutes (e.g. 60 = last 60 minutes).
  • Rate Cap — Maximum number of search actions allowed within that window.
  • Example: Window=60, Cap=25 → max 25 searches per hour.
  • The progress bar on each dashboard card shows current usage.
Quiet Hours

During quiet hours, automatic searches are skipped. This is useful to avoid activity during night hours or when bandwidth is limited.

  • Overnight ranges work: e.g. Start=23:00, End=06:00.
  • Force runs from the dashboard always bypass quiet hours.
  • Leave both fields empty to disable quiet hours entirely.
FAQ & Troubleshooting
Instance shows "offline"

Missingarr cannot reach the configured URL. Check that the IP/port is correct and that your *arr instance is running. If running in Docker, make sure the network is accessible.

Instance shows "error"

The connection works, but the API key is invalid. Find the correct key under Settings → General → API Key in your *arr instance.

Rate cap reached — searches stop mid-run

Increase the Rate Cap or widen the Rate Window. The cap resets automatically after the window expires.

Radarr v6 — "Invalid API key" even with correct key

Radarr v6 removed Basic Authentication. Missingarr uses the X-Api-Key header which is the correct method. Make sure the API key is copied exactly without leading/trailing spaces.
